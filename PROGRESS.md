# DragonGlareAlpha 進捗メモ

更新日: 2026-03-23

## 現状サマリー

- `DragonGlareAlpha` は `net10.0-windows` を対象にした WinForms 製のRPG風プロトタイプです。
- 画面遷移は「モード選択 → 言語選択 → 名前入力 → フィールド → バトル / ショップ」まで通っています。
- 2026-03-23 の更新で、責務分割リファクタリングと、バトル・ショップ・セーブ・テストの不足分をまとめて補強しました。

## 今回の主な更新

### 構成整理

- `Form1.cs` に集まっていた処理を、`UI/` 配下の partial class へ分割
  - `Form1.Audio.cs`
  - `Form1.Input.cs`
  - `Form1.Persistence.cs`
  - `Form1.Rendering.cs`
  - `Form1.Updates.cs`
  - `Form1.Helpers.cs`
- ゲームデータは `Data/`
- ドメインモデルは `Domain/`
- 保存処理は `Persistence/`
- ロジックサービスは `Services/`
- テストコードは `DragonGlareAlpha.Tests/`

### バトル

- 敵データを `EnemyDefinition` として定義
- ランダム遭遇で敵ステータスを持つバトルを開始
- プレイヤー攻撃、じゅもん、敵の反撃、逃走を実装
- 勝利 / 敗北 / 逃走の状態管理を追加
- 勝利時に経験値とゴールドを獲得
- レベルアップ時に HP / MP / ATK / DEF が成長
- 敗北時はHP/MP全快で開始地点へ戻り、所持金の一部を失う

### ショップ

- 商品データを `WeaponDefinition` として定義
- 購入時に所持金を消費してインベントリへ反映
- 強い武器を買った場合は自動装備
- 所持数、現在装備、ATK/DEF をUI上に表示

### セーブ / ロード

- `savegame.json` の保存対象を拡張
  - 言語
  - 名前
  - 座標
  - レベル
  - 経験値
  - HP / MP
  - 基礎ATK / DEF
  - ゴールド
  - 装備中の武器
  - 所持品
- 古い簡易セーブから読み込んだ場合も、欠けている値は既定値で補完
- フィールド移動、購入、戦闘結果反映後にも保存される構成へ変更

### テスト

- `xUnit` ベースのテストプロジェクトを追加
- 現在の自動テスト対象
  - バトル勝利判定
  - 経験値獲得とレベルアップ
  - 敗北時の復帰処理
  - ショップ購入と自動装備
  - セーブ / ロードの往復
